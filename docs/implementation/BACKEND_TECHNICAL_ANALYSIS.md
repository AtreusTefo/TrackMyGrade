#### **AdminValidator.cs** (Static Methods)

Location: `TrackMyGradeAPI/Application/Validators/AdminValidator.cs`

**NOTE:** Uses STATIC validation methods instead of AbstractValidator classes. This is inconsistent with other validators.

**Methods:**
- `ValidateCreateTeacher(AdminCreateTeacherDto)`
  - FirstName: NotNull, Length < 100
  - LastName: NotNull, Length < 100
  - Email: NotNull, Regex match (@"^[^@\s]+@[^@\s]+\.[^@\s]+$")
  - **Phone: !null && Regex match (@"^\+?[0-9\-\(\)\s]{7,}$")** ✓ Validates if present
  - Subject: Length <= 100
  - **Issue:** Static method, doesn't use CascadeMode

- `ValidateCreateStudent(AdminCreateStudentDto)`
  - FirstName/LastName: NotNull, Length < 100
  - Email: NotNull, Regex match
  - **Phone: !null && Length <= 20 && Regex match (@"^\+?[0-9\-\(\)\s]{7,}$")** ✓
  - OmangOrPassport: NotNull, Length <= 20
  - Grade: 1-12
  - TeacherId: > 0
  - **Issue:** Regex allows 7+ digits and special chars, contradicts DB CHECK (exactly 8 digits)

- `ValidateUpdateStudent(AdminUpdateStudentDto)`
  - Delegates to ValidateCreateStudent with mapped fields

- `ValidateCreateCourse(CreateCourseDto)`
  - Name: NotNull, Length <= 200
  - Code: NotNull, Length <= 20
  - Description: !null && Length <= 500

- `ValidateCreateClassGroup(CreateClassGroupDto)`
  - Name: NotNull, Length <= 100
  - GradeLevel: 1-12
  - CourseId: > 0
  - TeacherId: > 0

### 5.2 Soft Delete Implementation ✗

**Status:** NOT IMPLEMENTED

No `IsDeleted` flag on any entity. All deletes are hard deletes:
```csharp
_db.Teachers.Remove(teacher);
_db.SaveChanges();  // Hard delete
```

**Issue:** Audit trail captures deletion, but deleted data is permanently removed from tables. This violates the AGENTS.md requirement for soft deletes on grading entities.

---

### 5.3 Timestamp Management ✓

**UTC Timestamps on All Entities:**
- Teacher.ActivatedAt (DateTime?)
- Student.ActivatedAt (DateTime?)
- StudentEnrollment.EnrolledAt (DateTime)
- Assignment.CreatedAt (DateTime)
- AssignmentSubmission.SubmittedAt (DateTime)
- AuditLog.PerformedAt (DateTime)
- Admin.CreatedAt (DateTime)
- Admin.UpdatedAt (DateTime)

**Pattern:** `DateTime.UtcNow` used consistently in model defaults and service methods.

**Issue:** No UpdatedAt tracking on non-Admin entities (Teacher, Student, Course, ClassGroup, Assignment, AssignmentSubmission, StudentEnrollment).

---

### 5.4 Concurrency Control ✗

**Status:** NOT IMPLEMENTED

No `[Timestamp]` or `[ConcurrencyCheck]` attributes on any entity. EF6 has no concurrency detection for optimistic locking.

**Impact:** Concurrent updates to assignments, grades, or submissions could cause lost updates:
```csharp
// Two teachers grade same submission simultaneously
submission1.Score = 95;
submission2.Score = 87;
_db.SaveChanges();  // No conflict detection
```

---

### 5.5 Transaction Management ✗

**Status:** NOT IMPLEMENTED

No explicit `DbContext.Database.BeginTransaction()` usage found in any service.

**Grep Result:** Zero matches for transaction patterns in Services directory.

**Risk Scenarios:**
1. **CreateStudent + StudentEnrollment** (not atomic in current code)
2. **Assignment creation + auto-enrollment** (if implemented)
3. **Submission grading + AuditLog** (if audit log fails, grading succeeds, inconsistency)

**Current Pattern (Unsafe):**
```csharp
_db.Students.Add(student);
_db.SaveChanges();  // Point of failure #1

_auditLogService.LogCreate(...);  // Separate transaction
```

If audit log save fails, student is already committed to DB.
